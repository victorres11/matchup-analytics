Expected Behavior

  1. Click a test route above
  2. You should briefly see the 404.html page with debug info
  3. It should automatically redirect to the main app
  4. The main app should show the matchup information
Note: If you see a plain GitHub 404 page instead of our custom 404.html, there's a configuration issue.

Debugging

If routing isn't working:

  • Check the browser console for error messages
  • Verify that 404.html is in your repository root
  • Make sure GitHub Pages is configured to deploy from the main branch
  • Wait for GitHub Pages to redeploy after changes
